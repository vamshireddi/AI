Week 1 Linear Regression: Business Problem & Correlation

Business Problem Statement

Regression Problem:
Given historical data with features and outcomes, learn function to predict future continuous outcomes.
Real estate company has sold 500 houses. Features: square footage, bedrooms, location, year built. Can we predict price of new house?

Key Question: What mathematical relationship connects input features to output values?

Why Linear First? Simple, interpretable, fast. Great baseline.

Visualizing Relationships

Imagine height vs weight of people. Plot each person as dot: x-axis = height, y-axis = weight. You'll see cloud trending upward—taller people weigh more. That trend is the relationship we want to capture.

Scatter Plots & Correlation

A scatter plot shows (x, y) values visually. Look for patterns:

  • Positive trend: x increases, y increases → positive correlation
  • Negative trend: x increases, y decreases → negative correlation
  • No trend: Random cloud → no correlation (r ≈ 0)
Pearson's Correlation Coefficient (r):
Measures strength & direction of linear relationship. Range: -1 to +1.
r = 0.95 means almost perfect positive correlation. r = -0.3 means weak negative.
r = Σ(xᵢ - x̄)(yᵢ - ȳ) / √[Σ(xᵢ - x̄)² Σ(yᵢ - ȳ)²]
Numerator: co-variation. Denominator: normalizes to -1 to +1 range
⚡
Correlation ≠ Causation: High correlation doesn't mean x causes y. Could be coincidence or third variable causing both.

Week 1 Simple Linear Regression

Definition & Intuition

Simple Linear Regression:
Fit straight line through data points to model relationship between one input (x) and one output (y).
House price data: x = square footage, y = price. Draw best-fit line. Given new house area, predict price from line.
ŷ = β₀ + β₁x
ŷ = predicted y, β₀ = intercept (y when x=0), β₁ = slope (change in y per unit x)

Understanding the Coefficients

Intercept (β₀):
Y-value where regression line crosses y-axis (when x = 0).
House price model: intercept = $50,000. A house with 0 sq ft "costs" $50k (mathematical, not realistic).
Slope (β₁):
Change in predicted y for each 1-unit increase in x.
Slope = $100/sq ft. Each extra square foot adds $100 to predicted price.
💡
Interpretation matters: Always describe slope in context. "$100 per square foot" is actionable.

Best-Fit Line: Least Squares Method

Draw line through scattered points to pass as close as possible to all points. Least squares finds line minimizing total distance from points to line (squared).

The residual for each point is: eᵢ = yᵢ - ŷᵢ (actual minus predicted)

Least squares minimizes: Σ(eᵢ)² = Σ(yᵢ - ŷ)²

Least Squares Concept
Line is fitted to minimize sum of squared vertical distances from points to line.
🎯
Why squared errors? Penalizes large errors heavily. Prevents large misses from offsetting small hits.
# Simple Linear Regression with sklearn
from sklearn.linear_model import LinearRegression
import numpy as np

X = np.array([[100], [200], [300], [400], [500]])
y = np.array([10000, 20000, 30000, 40000, 50000])

model = LinearRegression()
model.fit(X, y)

y_pred = model.predict([[350]])

Week 1 Multiple Linear Regression

From Simple to Multiple

Multiple Linear Regression:
Use multiple features (x₁, x₂, ..., xₚ) to predict one target (y).
Predict house price using: square footage, bedrooms, bathrooms, year built, location. Each gets its own coefficient.
ŷ = β₀ + β₁x₁ + β₂x₂ + ... + βₚxₚ
β₀ = intercept, β₁...βₚ = partial slopes. Each βᵢ = change in y per unit xᵢ (holding others constant)
📊
"Holding others constant": β₁ = impact of x₁ after accounting for x₂, x₃, etc. That's the power of multiple regression!

Categorical Variables: One-Hot Encoding

Features like "location" (North, South, East, West) are categorical. Can't plug directly. Solution: one-hot encoding

Imagine checklist: "Is location North? (Yes/No). Is it South? (Yes/No). Is it East? (Yes/No)." Convert to 0/1 columns. House in North: [1,0,0]. House in South: [0,1,0].

The Dummy Variable Trap

If location has 4 categories, create only 3 dummy columns, not 4. Why?

  • If North=1, South=0, East=0, then West=0 (implied). Perfect multicollinearity if include all 4.
  • Sklearn's get_dummies() creates all 4. Use drop_first=True to drop one.
  • Same rule: k categories → k-1 dummies
Multicollinearity:
Predictor variables highly correlated with each other. Coefficients become unstable.
If you include all 4 dummy columns (North, South, East, West), they sum to 1 always. Singular matrix. Model breaks.
⚠️
Always: Check drop_first=True when using categorical variables with regression. Prevents dummy variable trap.
# Handling categorical variables
import pandas as pd

df = pd.DataFrame({
    'price': [100000, 200000, 150000],
    'location': ['North', 'South', 'North']
})

df_encoded = pd.get_dummies(df, drop_first=True)

Week 1 Regression Evaluation Metrics

R² (Coefficient of Determination)

R²:
Proportion of variance in target explained by predictors. Range: 0 to 1.
R² = 0.85 means model explains 85% of price variation. 15% unexplained.
R² = 1 - (SS_res / SS_tot)
SS_res = sum of squared residuals. SS_tot = total sum of squares.
⚠️
R² increases with more features, even useless ones! Use Adjusted R² when comparing models.
R²_adjusted = 1 - [(1 - R²) × (n - 1) / (n - p - 1)]
Penalizes each added variable. Use to compare models with different numbers of predictors.

Error Metrics: MAE & RMSE

MAE (Mean Absolute Error):
Average absolute prediction error. In same units as target.
House price model MAE = $5,000. On average, predictions off by $5,000.
MAE = (1/n) Σ|yᵢ - ŷᵢ|
Absolute value: ignores direction. Robust to outliers.
RMSE (Root Mean Squared Error):
Square root of average squared error. Penalizes large errors more.
RMSE = $8,000. Large errors get huge penalty.
RMSE = √[(1/n) Σ(yᵢ - ŷᵢ)²]
Squaring penalizes outliers. √ returns to original units.
MAE = "average difference" (robust). RMSE = "typical worst error" (penalizes outliers). RMSE always ≥ MAE.
🎯
Which to use? MAE if outliers don't matter. RMSE if large errors very costly. Use both.

Metrics Comparison Table

MetricFormulaRangeWhen to Use
R²1 - (SS_res/SS_tot)0 to 1 (higher better)Explain variance. Compare similar models.
Adj R²1 - [(1-R²)(n-1)/(n-p-1)]0 to 1 (higher better)Compare models with different features.
MAE(1/n)Σ|yᵢ - ŷᵢ|0 to ∞ (lower better)Interpretable. Robust to outliers.
RMSE√[(1/n)Σ(yᵢ - ŷᵢ)²]0 to ∞ (lower better)Penalize large errors. Fair comparison.
# Calculate all metrics with sklearn
from sklearn.metrics import r2_score, mean_absolute_error, mean_squared_error
import numpy as np

r2 = r2_score(y_test, y_pred)
mae = mean_absolute_error(y_test, y_pred)
rmse = np.sqrt(mean_squared_error(y_test, y_pred))

Week 2 Decision Trees: From Intuition to Implementation

Classification vs Regression Recap

Regression: Predict continuous values (price, temperature). Classification: Predict categories (spam/not, pass/fail).

Decision Trees work for both! This week focuses on classification trees.

Decision Tree:
Hierarchical model making decisions by recursively splitting data. Each split maximizes information gain or minimizes impurity.
Email spam detector: "Is word 'free' present? If yes, check word count. If > 100 words, likely spam." Tree of conditions.

Impurity Measures: Entropy & Gini

Entropy:
Measure of disorder/randomness. Max entropy = equal mix of all classes. Min entropy = all one class.
Dataset 50% spam, 50% ham: high entropy. Dataset 99% spam, 1% ham: low entropy.
Entropy = -Σ pᵢ log₂(pᵢ)
pᵢ = proportion of class i. Max = 1, min = 0.
Bag of colored balls. All same color = boring (low entropy). Half red, half blue = unpredictable (high entropy). Entropy measures that unpredictability.
Gini Impurity:
Probability of incorrectly classifying random point. Range 0 (pure) to 0.5 (balanced).
Dataset 80% class A, 20% class B: Gini = 1 - (0.8² + 0.2²) = 0.32.
Gini = 1 - Σ pᵢ²
Simpler math than entropy. Both measure impurity.
💡
Interpretation: Both = 0 means pure node. Both = max when balanced. Use to select best split.

Information Gain: Choosing the Best Split

Information Gain:
Reduction in impurity after split. Tree chooses split maximizing information gain.
Before split: entropy 0.9. After split: left (0.3), right (0.4). Big reduction = high IG.
Information Gain = Entropy(parent) - Σ [weighted Entropy(children)]
Weighted = proportion in each child. Higher IG = better split.
Tree Building Process
Start with all data
↓
Try all possible splits
↓
Pick split with max IG
↓
Recursively repeat
↓
Stop when pure or min samples reached
🎯
Greedy approach: Best local split at each node. Doesn't guarantee globally optimal tree, but fast.

Hands-On: Build & Visualize

# Decision Tree Classifier
from sklearn.tree import DecisionTreeClassifier, plot_tree
import matplotlib.pyplot as plt

model = DecisionTreeClassifier(max_depth=5, random_state=42)
model.fit(X_train, y_train)

plot_tree(model, feature_names=['age', 'income'], filled=True)
plt.show()

accuracy = model.score(X_test, y_test)

Week 2 Overfitting & Pruning

The Overfitting Problem

Overfitting:
Model learns training data too well, including noise. Poor test performance.
Decision tree with no depth limit grows very deep. Fits training perfectly but fails on test. Tree memorized.
Student memorizes exact exam questions & answers, but can't solve new problems. Memorization vs understanding.
⚠️
Training accuracy ≠ Test accuracy. High training, low test = overfitting. Model doesn't generalize.

Pre-Pruning: Prevent Growth

Pre-Pruning:
Stop tree growth early. Control via hyperparameters.

Key hyperparameters in sklearn:

  • max_depth: Max levels tree can have. Lower = simpler, less overfitting.
  • min_samples_split: Min samples before split. Default = 2. Higher = less splits.
  • min_samples_leaf: Min samples in leaf. Prevents tiny leaves.
  • max_leaf_nodes: Max leaf nodes. Limits tree size.
🎯
Start simple: max_depth=5, then tune. Too restrictive = underfitting. Too loose = overfitting.
# Pre-pruning with hyperparameters
model = DecisionTreeClassifier(
    max_depth=5,
    min_samples_split=10,
    min_samples_leaf=4,
    random_state=42
)

Post-Pruning: Remove Weak Branches

Post-Pruning:
Grow full tree, then remove branches not improving validation performance.
Tree has 20 nodes. Remove node 15. If validation accuracy stays same or improves, prune it.

Steps:

1
Grow full tree with minimal restrictions
2
For each internal node, calculate cost if pruned vs kept
3
Prune weakest nodes (lowest validation impact)
4
Repeat until cost-benefit no longer favors pruning
💡
Sklearn note: No built-in post-pruning. Use pre-pruning or cost_complexity_pruning_path.

Bias-Variance Tradeoff

🎯
Goal: Maximize test accuracy. Monitor both train & test. If test plateaus while train rises = overfitting.

Week 3 K-Means Clustering

Unsupervised Learning Intro

Unsupervised Learning:
No labels/targets. Goal: discover hidden structure or patterns in data.
Customer data (age, spending, region) with no target. Goal: segment customers into groups with similar traits.
Supervised = labeled photos, sort by species. Unsupervised = pile of unlabeled photos. You find natural groups on your own.
Clustering:
Partitioning data into groups where within-group similarity is high, between-group similarity is low.
Customer clustering: High earners grouped, low earners separately. Different marketing per group.

K-Means Algorithm

K-Means:
Partitions data into k clusters by iteratively minimizing within-cluster sum of squared distances.

Algorithm steps:

1
Initialize: Randomly place k centroids in data space
2
Assign: Each point goes to nearest centroid (forms k clusters)
3
Update: Move each centroid to mean of its assigned points
4
Repeat: Steps 2-3 until centroids don't move
K-Means Process (Simplified)
Random Start → Assign Points → Move Centroids → Assign Again → ... → Converged
WCSS (Within-Cluster Sum of Squares) = Σ Σ ||xᵢ - μₖ||²
Distance from each point to centroid squared, summed. K-Means minimizes WCSS.
⚡
K-Means assumes: Spherical clusters of similar size. Long/elliptical clusters may not work well.
# K-Means with sklearn
from sklearn.cluster import KMeans

model = KMeans(n_clusters=3, init='k-means++', random_state=42)
clusters = model.fit_predict(X)
wcss = model.inertia_

Choosing K: Elbow Method & Silhouette

Elbow Method:
Plot WCSS vs K. Look for "elbow" where WCSS stops decreasing sharply. That's optimal k.
WCSS: K=1→100, K=2→40, K=3→15, K=4→14. Elbow at K=3 (big drop before, small after).
Silhouette Score:
Measures clustering quality. Range -1 to +1. +1 = perfect.
Silhouette = 0.65 = good clustering. Silhouette = 0.2 = weak, overlapping.
Silhouette(point) = (b - a) / max(a, b)
a = avg distance to same cluster. b = avg distance to nearest other cluster.
# Elbow method & Silhouette analysis
from sklearn.metrics import silhouette_score

wcss_list = []
for k in range(2, 11):
    km = KMeans(n_clusters=k, random_state=42)
    km.fit(X)
    wcss_list.append(km.inertia_)
⚠️
K-Means++: Spread initial centroids far apart. Converges faster. Use init='k-means++'

Week 3 Hierarchical Clustering

Agglomerative (Bottom-Up) Approach

Hierarchical Clustering:
Builds tree of clusters (dendrogram). Shows nested relationships at different distance thresholds.
Agglomerative: Start with each point as separate cluster. Merge closest pairs until all in one cluster.
Organize people into teams. Start with everyone alone. Pair closest friends. Merge pairs that are close. Eventually all together. At any height, cut dendrogram to get k clusters.

Agglomerative (Bottom-Up):

1
Start: Each point is its own cluster (n clusters)
2
Find two nearest clusters. Merge them (n-1 clusters)
3
Repeat until all points in one cluster
4
Dendrogram shows merging history. Cut at height = desired k

Linkage Methods: How to Measure Distance

When merging clusters, how do you measure distance? Four common methods:

Linkage TypeDistance CalculationWhen to UseBias
SingleMin distance between any two pointsChain clustersTends to chain together
CompleteMax distance between any two pointsCompact clustersTends to produce spherical
AverageMean distance between all pairsBalanced approachCompromise
WardMinimizes within-cluster varianceMost common, good defaultSimilar-sized clusters
Dendrogram:
Tree diagram showing cluster merging history. Height = distance at merge. Horizontal line = cut threshold for k clusters.
Samples A,B merge at height 5. Samples C,D merge at height 8. At height 10, all together. Cut at height 7 → 2 clusters.
# Hierarchical clustering
from sklearn.cluster import AgglomerativeClustering
from scipy.cluster.hierarchy import dendrogram, linkage

Z = linkage(X, method='ward')
dendrogram(Z)

model = AgglomerativeClustering(n_clusters=3, linkage='ward')
labels = model.fit_predict(X)
💡
Ward is default: Minimizes within-cluster variance. Similar to K-Means. Good starting point.

Dendrograms & Interpretation

Dendrogram is like evolutionary tree. Height = "distance" or "time since common ancestor." Horizontal line = you cutting tree, determining clusters.

Reading dendrograms:

  • Y-axis = distance at merge. Higher = more different clusters.
  • Vertical lines = points or clusters. Horizontal = merge event.
  • Cut dendrogram at height to get k clusters. Higher = fewer clusters.
  • Tall jumps indicate natural boundaries.
🎯
Advantage over K-Means: No need to specify k upfront. Visualize dendrogram first. Handles non-spherical clusters better.

Week 3 DBSCAN: Density-Based Clustering

Why DBSCAN?

K-Means & Hierarchical assume globular (spherical) clusters. Real data often has irregular shapes.

DBSCAN (Density-Based Spatial Clustering):
Groups points based on density. Finds clusters of any shape. Labels low-density points as outliers.
Credit card fraud: Normal transactions (high density). Frauds (isolated, low density). DBSCAN finds frauds as outliers.
Groups of people in room. Densely packed groups = clusters. Isolated people = outliers/noise. DBSCAN finds dense groups, ignores sparse.

Key advantage: Detects outliers. K-Means forces every point into cluster. DBSCAN can reject points as noise.

DBSCAN Algorithm & Parameters

Core Points, Border Points, Noise:
Core = at least min_samples neighbors within eps. Border = neighbor of core. Noise = neither.

Key parameters:

  • eps (ε): Radius of neighborhood. Too small = many noise. Too large = merges clusters.
  • min_samples: Min neighbors to be core point. Default = 2*dimensions. High = more noise.

Algorithm:

1
For each point, find all neighbors within eps
2
If ≥ min_samples neighbors, mark as core point
3
Core points within eps → same cluster
4
Border points added to nearest core's cluster. Noise stays alone.
⚡
No k parameter: DBSCAN discovers clusters. eps & min_samples control granularity.
# DBSCAN clustering
from sklearn.cluster import DBSCAN
from sklearn.preprocessing import StandardScaler

X_scaled = StandardScaler().fit_transform(X)

model = DBSCAN(eps=0.3, min_samples=5)
labels = model.fit_predict(X_scaled)

n_clusters = len(set(labels))
n_noise = list(labels).count(-1)

Common Mistakes & How to Avoid Them

Mistake 1: Forgetting drop_first=True with Categorical Variables
Error: model.fit() crashes with "singular matrix" or coefficients are NaN.
Fix: Always use pd.get_dummies(df, drop_first=True). k categories → k-1 dummies.
Mistake 2: Comparing Models Using Only R² (High Bias)
Error: Add useless feature. R² increases (always does with more features). You think improved, but didn't.
Fix: Use Adjusted R² for comparison. Check test set performance.
Mistake 3: Not Scaling Features for K-Means
Error: Feature "income" (0-1,000,000) dominates "age" (0-100). K-Means uses Euclidean, sensitive to scale.
Fix: StandardScaler or MinMaxScaler before clustering. StandardScaler().fit_transform(X)
Mistake 4: Decision Tree: Not Preventing Overfitting
Error: Tree grows unbounded. Train = 100%, test = 60%. Memorized noise.
Fix: Use pre-pruning: max_depth, min_samples_split, min_samples_leaf. Monitor test accuracy.
Mistake 5: Choosing K in K-Means by Elbow Alone
Error: Elbow method is heuristic. Denbow not always clear. Pick K=4 but K=5 better.
Fix: Use both elbow AND silhouette score. Try multiple K. Cross-validate.
Mistake 6: Confusing Correlation with Causation
Error: Correlation = 0.9 between hours studied and test score. "More studying causes higher scores." Maybe both from student ability.
Fix: Correlation shows association only. Prove causation with experiments or domain logic.
Mistake 7: K-Means on Non-Spherical Data
Error: Data has elongated/crescent clusters. K-Means assumes spherical. Bad clusters.
Fix: Use DBSCAN or hierarchical clustering. Or transform features (e.g., PCA) first.
Mistake 8: Ignoring Interpretation of Regression Coefficients
Error: β₁ = 0.0034. You say "slope is 0.0034" without context. Not actionable.
Fix: Describe coefficient in business context. "Each year of experience adds $3,400 to salary." Check units!

Frequently Asked Questions & Interview Prep

Regression Interview Questions

Q1: Explain the difference between R² and Adjusted R².
R² increases with any added variable. Adjusted R² penalizes extra variables via (n-1)/(n-p-1) factor. Use Adjusted R² for fair model comparison.
Model A: R²=0.85 with 5 features. Model B: R²=0.87 with 20 features. Adjusted R² A=0.84, B=0.80. Model A is better despite lower R²!
Q2: When should you use MAE vs RMSE?
MAE = robust to outliers (doesn't penalize huge errors as much). RMSE = sensitive to outliers (penalizes large errors heavily).
Predictions [1, 2, 3, 1000]: MAE = 251.5, RMSE = 500.1. If outlier matters (like hospital readmissions), use RMSE.
Q3: Your model has multicollinearity. What happens & how to fix?
Coefficients become unstable, interpretations unreliable, predictions OK. Check VIF. If VIF > 5-10, remove or combine correlated features.
Income & wealth highly correlated (r=0.95). Both coefficients ±wildly. Remove one or use PCA. Problem fixed.
Q4: Why scale features for regression? Is it necessary?
Not necessary for Linear Regression (invariant to scaling). BUT necessary for: gradient descent (converges faster), regularization (Ridge/Lasso), distance-based methods.
Income (0-1M) vs years employed (0-50). If using Ridge, scaling prevents income dominating penalty. Always scale for consistency.

Decision Trees & Ensemble Methods Interview Questions

Q5: Explain overfitting in decision trees and how to prevent it.
Tree grows deep, memorizes training data including noise. Train accuracy high, test accuracy low. Prevent via: max_depth, min_samples_split, min_samples_leaf, post-pruning.
Unrestricted tree: 100 nodes, train=99%, test=60%. Limited tree (max_depth=5): train=85%, test=82%. Gap reduced = less overfitting.
Q6: What is information gain and how does tree selection use it?
IG = reduction in impurity after split. Tree greedily picks split maximizing IG at each node. Doesn't guarantee globally optimal tree.
Before split: Entropy=0.8. After split A: Entropy=0.3, 0.2 (IG=0.5). After split B: Entropy=0.5, 0.3 (IG=0.3). Choose A (higher IG).
Q7: Gini vs Entropy - which is better?
Both measure impurity, produce similar trees. Gini simpler math (no log). Entropy more information-theoretic. sklearn defaults to Gini. Choose either.
For classification: both work. Gini slightly faster. Entropy slightly more interpretable. Performance difference negligible.
Q8: How do you interpret feature importance from tree?
Feature importance = total reduction in impurity across all splits using that feature. Higher = more influential in predictions.
Credit score: 0.45, Income: 0.35, Employment: 0.20. Credit score drives model most. First split often uses most important feature.

Clustering Interview Questions

Q9: How do you choose k in K-Means?
No perfect answer. Elbow method (plot WCSS vs k), Silhouette score (higher = better), domain knowledge. Combine methods.
Elbow suggests k=3. Silhouette: k=2→0.45, k=3→0.52, k=4→0.50. Choose k=3. Domain: 3 customer segments makes business sense.
Q10: Why is K-Means++ better than random initialization?
K-Means++ spreads initial centroids far apart. Converges faster & finds better solutions. Random init may get stuck in local minima.
Random init: converges in 15 iterations. K-Means++: converges in 4 iterations. Same WCSS, much faster. Use init='k-means++' always.
Q11: When should you use DBSCAN instead of K-Means?
DBSCAN when: (1) clusters non-spherical, (2) outliers important, (3) k unknown. K-Means when: (1) spherical clusters, (2) fast needed, (3) k specified.
Fraud detection: K-Means forces every transaction into cluster. DBSCAN labels anomalies as -1 (noise) = frauds detected! Use DBSCAN.
Q12: What does Silhouette score measure? How to interpret?
Silhouette = how similar point is to its cluster vs nearest other cluster. Range -1 to +1. -1 = wrong cluster, 0 = ambiguous, +1 = perfect.
Silhouette = 0.65 = good clustering. Silhouette = -0.1 = some points in wrong clusters. Silhouette = 0.2 = weak, overlapping clusters.

Model Evaluation Best Practices Checklist

Before deploying any model, verify:

  • Data quality: Check for missing values, outliers, class imbalance
  • Train/test split: Proper stratification for classification, time-series aware for temporal data
  • No data leakage: Preprocessing fit on train only, target info never in features
  • Baseline comparison: Model beats simple baseline (mean predictor, dummy classifier)
  • Multiple metrics: Don't rely on single metric (R², accuracy). Check MAE, RMSE, precision/recall for classification
  • Cross-validation: 5-fold CV to ensure stable performance across different data splits
  • Feature importance: Verify important features make business sense (not random noise)
  • Residual analysis (regression): Plot residuals. Should be random, normally distributed, homoscedastic (constant variance)
  • Confusion matrix (classification): Understand false positives vs false negatives impact for your problem
  • Sensitivity analysis: Small changes in hyperparameters shouldn't drastically change results
Deploying untested model = flying plane without pre-flight check. Crashes guaranteed. Checklist prevents catastrophe.

Model Selection Decision Tree

How to choose between algorithms?

1
Is target continuous or categorical? Continuous → Regression (linear, tree-based). Categorical → Classification (tree, etc.).
2
Do you have labels? Yes → Supervised (regression, classification). No → Unsupervised (clustering).
3
Supervised regression: Start with Linear. If non-linear, try Tree/Polynomial.
4
Supervised classification: Tree is good baseline. Fast, interpretable.
5
Unsupervised clustering: K-Means if spherical, DBSCAN if arbitrary shape, Hierarchical to explore.

Regression (simple) Linear Regression Fast, interpretable, linear baseline Polynomial, Ridge, Tree
Regression (complex) Decision Tree Captures non-linearity, handles mixed types Random Forest, Gradient Boosting
Classification Decision Tree Interpretable, handles mixed features, no scaling needed Logistic, SVM, Random Forest
Clustering (spherical) K-Means Fast, scalable, simple Hierarchical with different linkage
Clustering (arbitrary shape) DBSCAN No need for k, handles outliers, finds non-convex shapes Hierarchical Clustering

Advanced Regression Topics: Ridge & Lasso Regularization

Problem: Complex models overfit. Solution: Penalize large coefficients via regularization.

Types of regularization:

Ridge (L2) Regression:
Adds penalty = λ × Σ(βᵢ²) to loss function. Shrinks all coefficients toward zero, but never exactly zero.
High λ: All coefficients approach 0, model simpler, may underfit. Low λ: Coefficients unconstrained, may overfit. Find balance via CV.
Lasso (L1) Regression:
Adds penalty = λ × Σ|βᵢ| to loss function. Shrinks some coefficients exactly to zero (feature selection!).
If feature unhelpful, Lasso sets its coefficient = 0 (removes it). Ridge keeps it but small. Lasso = automatic feature selection.
# Ridge vs Lasso vs Linear
from sklearn.linear_model import LinearRegression, Ridge, Lasso
from sklearn.model_selection import GridSearchCV

Feature Engineering & Preprocessing Techniques

Good features = good models. Garbage in, garbage out!

Feature engineering tactics:

  • Domain knowledge: Create new features from existing ones based on business logic (e.g., age² for non-linear effects)
  • Polynomial features: x² , x³, x₁×x₂ to capture interactions
  • Log transformation: If data skewed, log(x) normalizes distribution
  • Binning/Discretization: Convert continuous to categorical (age groups: 0-18, 18-35, 35-65, 65+)
  • Normalization: Scale to 0-1 range (MinMaxScaler). Useful for neural networks.
  • Standardization: Scale to mean=0, std=1 (StandardScaler). Useful for distance-based algorithms.
  • Target encoding: For categorical with target correlation, encode as target mean within that category
  • Dropping low-variance features: Features with zero variance (same value) add no info, remove

FAQ Responses & Clarifications

Q: What is the intercept in linear regression? Why is it sometimes weird?
The intercept β₀ is the predicted y value when all predictors = 0. Mathematically necessary, but often not interpretable in context.
House price model: intercept = $50,000. A 0-square-foot house doesn't exist! Intercept is just the mathematical anchor point for the line. Never interpret intercepts literally unless x=0 makes sense.
Q: My dataset has one categorical with 100 categories. Should I create 100 dummies?
No! Create 99 dummies (drop_first=True). 100 dummies → perfect multicollinearity → singular matrix.
City variable: NYC, LA, Chicago, ..., (100 cities). Create dummies for 99 cities. If all 99 = 0, then it's the 100th city (implied).
Q: My tree has train accuracy 98%, test accuracy 72%. How do I fix it?
Classic overfitting. Tree memorized training data including noise. Solution: reduce complexity via hyperparameters.
Reduce max_depth from unlimited to 5. Increase min_samples_split from 2 to 20. Increase min_samples_leaf from 1 to 10. Recheck train/test accuracy gap.
Q: K-Means found k=3 clusters, but elbow looks ambiguous. Should I trust silhouette?
Elbow method is heuristic. Use silhouette as tiebreaker. Higher silhouette score = better separated clusters.
Silhouette scores: k=2→0.45, k=3→0.52, k=4→0.50. Choose k=3 (highest silhouette). Confirms elbow ambiguity resolves toward k=3.
Q: Can I use categorical variables directly in K-Means or Decision Tree?
Decision trees handle categorical directly (split on categories). K-Means cannot (uses Euclidean distance). Encode categorical as dummies or ordinal for K-Means.
Color variable (Red, Blue, Green): DT uses as-is. K-Means needs [1,0,0], [0,1,0], [0,0,1]. Or ordinal: Red=0, Blue=1, Green=2 (if order matters).
Q: How do I handle missing values in regression/clustering?
Options: (1) Drop rows with missing (lose data). (2) Impute with mean/median (assumes MCAR). (3) Impute with KNN/iterative (more sophisticated).
10% missing values: impute with median to keep 90% of data. 50% missing: likely drop rows or features. Use sklearn.impute.SimpleImputer.
Q: Should I scale features before Decision Tree? Before K-Means?
DT: No (tree-based, not distance-based). K-Means: Yes (uses Euclidean distance). Hierarchical/DBSCAN: Yes (distance-based). Regression: No (invariant), but yes for Ridge/Lasso.
DT with income (0-1M) and age (0-100): both work fine, tree finds optimal splits. K-Means: income dominates (larger scale). Must scale both to equal importance.
Q: What is train/test split ratio? Does it matter?
Common: 70/30, 80/20, or 60/20/20 (include validation). Depends on data size. Small dataset: 80/20 (preserve training data). Large dataset: 70/30 OK.
1000 samples: 70% train=700, 30% test=300 (enough for reliable evaluation). 100k samples: 80% train=80k, 20% test=20k (still robust).
Q: How do I prevent data leakage in preprocessing?
CRITICAL: fit all preprocessors (scaler, imputer) on TRAINING only. Transform test using training statistics. If fit on test, information leaks from test into training.
WRONG: Scaler.fit(all_data). CORRECT: Scaler.fit(X_train), then Scaler.transform(X_train), Scaler.transform(X_test). Test statistics NEVER used in training.
