🔤 1.2 Variables & Data Types

Week 1
Variable
A named container that stores a value in memory. In Python, you create a variable simply by assigning a value: x = 10. No need to declare the type — Python figures it out automatically (dynamic typing).
A labeled jar in your kitchen. The label says "sugar" (variable name), and inside is the actual sugar (the value). You can empty it and fill it with salt later — the jar does not care what is inside.
Dynamic Typing
Python determines the type of a variable at runtime based on the value assigned. You can reassign a variable to a completely different type without errors. Use type(x) to check the current type.
A parking spot that accepts any vehicle — car, motorcycle, or truck. The spot does not need a sign saying "cars only."
Type Conversion (Casting)
Explicitly changing a value from one type to another using functions like int(), float(), str(), bool(). Essential when combining different types — you cannot concatenate a string and an integer directly.
Converting currency — changing US dollars to Euros. The value stays the same in principle, but the format changes so it can be used in a different context.
Variables are like phone contacts. The contact name ("Mom") is the variable name, and the phone number (555-1234) is the value. You can update the number anytime, and the name still works. You can even change "Mom" to store an email address instead of a phone number — Python does not mind.

int32 vs int64 (FAQ #8)

When working with NumPy and Pandas, you will encounter int32 and int64:

  • int32: Uses 4 bytes (32 bits) of memory. Range: -2,147,483,648 to 2,147,483,647
  • int64: Uses 8 bytes (64 bits) of memory. Range: approximately -9.2 × 10¹⁸ to 9.2 × 10¹⁸

For most data science work, int64 is the default and perfectly fine. Only optimize to int32 when memory is a constraint with large datasets.

📦 1.3 Data Structures: Lists, Tuples, Dictionaries

Week 1
List
A mutable, ordered collection of items enclosed in square brackets []. Lists can hold mixed types (integers, strings, booleans, even other lists). Created with list() or [].
A shopping list on paper — you can add items, cross them off, rearrange them, and they stay in order.
Tuple
An immutable, ordered collection enclosed in parentheses (). Once created, you cannot add, remove, or modify elements. Created with tuple() or ().
A sealed time capsule — once you put items in and seal it, you can look at the contents but cannot change them.
Dictionary
A mutable, unordered collection of key-value pairs enclosed in curly braces {}. Access values by their unique key, not by position. Created with dict() or {}.
A real dictionary book — you look up a word (key) to find its definition (value). You do not search by page number.
A list is a shelf with numbered slots — items stay in order, you can swap them. A tuple is a display case with a glass cover — look but do not touch. A dictionary is a filing cabinet — each drawer has a label (key), inside is the content (value).

🔀 1.4 Control Flow: Conditionals & Loops

Week 1
Conditional Statement (if/elif/else)
A control structure that executes different blocks of code based on whether a condition is True or False. Python uses indentation (4 spaces) to define code blocks — no curly braces like other languages.
A traffic light — if green, go. Elif (else if) yellow, slow down. Else (red), stop. Only one action happens based on the current state.
For Loop
Iterates over a sequence (list, tuple, string, range) executing a block of code for each element. The variable i (or any name you choose) takes on each value in the sequence, one at a time (FAQ #13).
A postal worker delivering mail — they visit each house on the street in order, drop off the mail, then move to the next house.
While Loop
Repeats a block of code as long as a condition remains True. Be careful — if the condition never becomes False, you get an infinite loop! Always make sure the condition will eventually change (FAQ #12).
Filling a bathtub — keep the water running WHILE the tub is not full. Once it is full, stop. If you forget to check, the water overflows (infinite loop).
Control flow is like a choose-your-own-adventure book. At each decision point (if), you pick a path. For loops know how many times to repeat; while loops keep going until a condition changes.

↓

# ============ CONDITIONALS ============
score = 85

if score >= 90:
    print("Grade: A")
elif score >= 80:
    print("Grade: B")    # This runs! score=85 >= 80
elif score >= 70:
    print("Grade: C")
else:
    print("Grade: F")

# Logical operators: and, or, not
age = 25
has_license = True
if age >= 18 and has_license:
    print("Can drive")

# ============ FOR LOOP ============
# Iterate over a list
fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(f"I like {fruit}")

# Using range(): range(start, stop, step) — FAQ #13
for i in range(0, 10, 2):   # 0, 2, 4, 6, 8
    print(i, end=" ")

# Nested loop with enumerate
for index, fruit in enumerate(fruits):
    print(f"{index}: {fruit}")   # 0: apple, 1: banana, 2: cherry

# ============ WHILE LOOP ============
count = 0
while count < 5:       # FAQ #12: runs while condition is True
    print(count)
    count += 1         # Don't forget this! Otherwise infinite loop

# break and continue
for i in range(10):
    if i == 3: continue   # Skip 3
    if i == 7: break      # Stop at 7
    print(i, end=" ")       # Output: 0 1 2 4 5 6

For Loop vs While Loop

For Loop — Use When:
  • You know how many iterations
  • Iterating over a sequence
  • Processing each item in a collection
  • Using range() for counted loops
While Loop — Use When:
  • Unknown number of iterations
  • Waiting for a condition to change
  • User input validation
  • Running until a flag is set

⚙️ 1.5 Functions & List Comprehension

Week 1
Function
A reusable block of code that performs a specific task. Defined with the def keyword, it can accept inputs (parameters) and return outputs. Functions eliminate code duplication and make programs modular.
A coffee machine — you put in water and beans (inputs), press a button (call the function), and get coffee (output). You do not need to know how the machine works internally to use it.
Return Statement
Sends a value back from a function to the caller. Without return, the function returns None by default. A function can return multiple values as a tuple: return a, b.
A vending machine — you insert money (arguments), the machine processes your selection, and returns the snack to you. If you just press buttons without inserting money, you get nothing (None).
Lambda Function
An anonymous, one-line function defined with the lambda keyword. Syntax: lambda arguments: expression. Best for simple operations where defining a full function would be overkill.
A sticky note with a quick formula — instead of writing a full instruction manual, you jot down "multiply by 2" on a Post-it and stick it where needed.
List Comprehension
A concise, Pythonic way to create lists in a single line. Syntax: [expression for item in iterable if condition]. It replaces multi-line for loops with a compact, readable expression.
A factory assembly line — raw materials (iterable) come in, each gets processed (expression), and defective ones are filtered out (condition). The output is a list of finished products.
Functions are recipes in a cookbook — define once, call by name whenever needed. Lambda functions are quick recipes on a napkin. List comprehensions are speed-cooking: make 10 dishes at once in one instruction.
# ============ DEFINING FUNCTIONS ============
def greet(name):
    """Greet a person by name."""
    return f"Hello, {name}!"

print(greet("Vamshi"))   # "Hello, Vamshi!"

# Default parameters
def power(base, exp=2):
    return base ** exp

print(power(3))       # 9 (uses default exp=2)
print(power(3, 3))    # 27 (exp=3)

# Multiple return values
def min_max(numbers):
    return min(numbers), max(numbers)

lo, hi = min_max([4, 2, 9, 1])
print(f"Min: {lo}, Max: {hi}")  # Min: 1, Max: 9

# ============ LAMBDA FUNCTIONS ============
double = lambda x: x * 2
print(double(5))     # 10

# Lambda with sorted
students = [("Alice", 85), ("Bob", 92), ("Charlie", 78)]
by_score = sorted(students, key=lambda s: s[1], reverse=True)
# [('Bob', 92), ('Alice', 85), ('Charlie', 78)]

# ============ LIST COMPREHENSION ============
# Instead of:
squares = []
for x in range(10):
    squares.append(x**2)

# Write this:
squares = [x**2 for x in range(10)]
# [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]

# With condition: only even squares
even_sq = [x**2 for x in range(10) if x % 2 == 0]
# [0, 4, 16, 36, 64]

# Nested: flatten 2D list
flat = [n for row in [[1,2],[3,4]] for n in row]  # [1,2,3,4]

🔢 2.1 NumPy: Numerical Computing

Week 2
NumPy (Numerical Python)
A fundamental Python library for numerical computing. Its core object is the ndarray (N-dimensional array), which enables fast mathematical operations on large datasets. NumPy is the foundation upon which Pandas, scikit-learn, and most data science libraries are built.
A calculator that can handle millions of numbers at once. Instead of adding numbers one by one in a loop, NumPy adds entire arrays in a single operation — like having a million calculators working simultaneously.
Vectorized Operations
Operations that apply to entire arrays at once, without explicit loops. NumPy executes these in optimized C code, making them 10-100x faster than Python loops. Examples: arr * 2, arr1 + arr2, np.sqrt(arr).
A factory production line that stamps all parts at once, vs. a craftsman filing each part individually.
Broadcasting
NumPy's ability to perform operations on arrays of different shapes by automatically expanding the smaller array to match. For example, adding a scalar to an array or adding a 1D array to each row of a 2D array.
Giving a $5 raise to every employee in a company. You state the raise once, and it "broadcasts" to everyone's salary — you do not need to write each person's new salary individually.
Think of Python lists as a regular backpack — flexible, can hold anything, but slow to search through. NumPy arrays are like an organized toolbox — everything has a fixed type and position, so finding and using tools is lightning fast. The trade-off: the toolbox only holds one type of item (all integers, all floats, etc.), but it is extremely efficient.

import numpy as np

# Creating arrays
arr = np.array([1, 2, 3, 4, 5])
zeros = np.zeros((3, 4))        # 3x4 matrix of zeros
ones = np.ones((2, 3))          # 2x3 matrix of ones
rng = np.arange(0, 10, 2)      # [0, 2, 4, 6, 8]
lin = np.linspace(0, 1, 5)     # [0, 0.25, 0.5, 0.75, 1.0]

# Array properties
print(arr.shape)     # (5,) — dimensions
print(arr.dtype)     # int64 — data type
print(arr.ndim)      # 1 — number of dimensions

# Vectorized operations — no loops needed!
print(arr * 2)       # [2, 4, 6, 8, 10]
print(arr ** 2)      # [1, 4, 9, 16, 25]
print(np.sqrt(arr))  # [1.0, 1.41, 1.73, 2.0, 2.24]

# Statistical functions
print(arr.mean())    # 3.0
print(arr.std())     # 1.414
print(arr.sum())     # 15
print(arr.min())     # 1

Missing Values (NaN)
Entries in your dataset that have no value. Represented as NaN (Not a Number) in Pandas. Can occur due to data entry errors, sensor failures, or optional fields. Must be handled before analysis because they propagate through calculations.
Blank cells in a spreadsheet — some customers did not fill in their phone number. You cannot compute an "average phone number," so you must decide: fill in a default, remove those rows, or ignore that column.
Data Type Conversion
Changing the data type of a column to the appropriate type. Common conversions: string to datetime, object to category, string to numeric. Use df['col'].astype() or pd.to_datetime(), pd.to_numeric().
Converting currency strings like "$1,500.00" to the number 1500.0 so you can do math with them.
Data cleaning is like preparing ingredients before cooking. You wash vegetables (remove duplicates), peel and chop them (type conversion), and check for any rotten pieces (missing values). No chef starts cooking with dirty, unprepped ingredients — and no data scientist starts analyzing dirty data.

🔍 3.2 EDA Techniques & Statistics

Week 3
Exploratory Data Analysis (EDA)
A systematic approach to investigating datasets using summary statistics and visualizations before formal modeling. The goal is to discover patterns, spot anomalies, test hypotheses, and check assumptions. John Tukey coined the term in 1977.
A detective examining a crime scene — before forming theories, you observe everything: what is there, what is missing, what looks unusual, what patterns emerge.
Outlier
A data point that lies significantly outside the overall pattern of the data. Can be detected using the IQR method (below Q1-1.5×IQR or above Q3+1.5×IQR) or Z-score (more than 3 standard deviations from the mean). May indicate errors, fraud, or genuinely extreme values.
In a classroom of 30 students with ages 18-22, one "student" listed as age 97 is clearly an outlier — likely a data entry error.
Correlation
A statistical measure (-1 to +1) of the linear relationship between two variables. +1 means perfect positive correlation, -1 means perfect negative, 0 means no linear relationship. Calculated with df.corr(). Remember: correlation does not imply causation.
Ice cream sales and drowning deaths are positively correlated — not because ice cream causes drowning, but because both increase in summer (confounding variable: hot weather).
EDA is like a doctor doing a checkup — take vital signs (summary stats), look at X-rays (visualizations), check for symptoms (outliers), and run blood tests (correlations) before diagnosing (modeling).

EDA Checklist

1
Shape & Structure: df.shape, df.info(), df.dtypes — How big is the data? What types are the columns?
2
Summary Statistics: df.describe() — Mean, median, min, max, quartiles for every numerical column.
3
Missing Values: df.isnull().sum() — Which columns have gaps? How much data is missing?
4
Univariate Analysis: Histograms and box plots for each variable individually — What does each variable look like?
5
Bivariate Analysis: Scatter plots, grouped bar charts — How do variables relate to each other?
6
Correlation Analysis: sns.heatmap(df.corr()) — Which variables move together?
7
Outlier Detection: IQR method or Z-scores — Are there extreme values that need attention?

Statistical Measures Quick Reference

Central Tendency
Mean: df["col"].mean() — average, sensitive to outliers
Median: df["col"].median() — middle value, robust to outliers
Mode: df["col"].mode() — most frequent value
Spread / Dispersion
Std Dev: df["col"].std() — average distance from mean
Variance: df["col"].var() — std dev squared
IQR: Q3 - Q1 — spread of middle 50%
Range: max - min — total spread
Shape
Skewness: df["col"].skew()
0 = symmetric, >0 = right-skewed, <0 = left-skewed
Kurtosis: df["col"].kurtosis()
Measures tail heaviness (outlier-proneness)
Outlier Detection: IQR
Q1 = 25th percentile
Q3 = 75th percentile
IQR = Q3 - Q1
Lower fence = Q1 - 1.5 × IQR
Upper fence = Q3 + 1.5 × IQR 
# Outlier detection using IQR method
Q1 = df["Salary"].quantile(0.25)
Q3 = df["Salary"].quantile(0.75)
IQR = Q3 - Q1

lower_bound = Q1 - 1.5 * IQR
upper_bound = Q3 + 1.5 * IQR

# Filter outliers
outliers = df[(df["Salary"] < lower_bound) | (df["Salary"] > upper_bound)]
print(f"Found {len(outliers)} outliers")

# Correlation matrix
corr_matrix = df.corr()
print(corr_matrix)

📝 4.1 Analyzing Text Data

Week 4
Regular Expression (Regex)
A powerful pattern-matching language for searching, extracting, and manipulating text. Python provides the re module. Common patterns: \d (digit), \w (word character), \s (whitespace), . (any character), * (0 or more), + (1 or more).
A metal detector on a beach — you define what you are looking for (pattern), and the regex engine scans through the entire text to find all matches.
Tokenization
The process of splitting text into individual units (tokens), typically words or sentences. The first step in most text analysis pipelines. Can be done with .split() for simple cases or with NLP libraries for complex text.
Breaking a sentence into individual words — like taking apart a LEGO structure into its individual bricks before sorting and counting them.
Stop Words
Common words (the, is, at, which, on) that carry little meaningful information and are typically removed during text preprocessing. Removing them reduces noise and improves analysis quality.
In a book review, words like "the," "is," and "was" appear everywhere but tell you nothing about whether the reviewer liked the book. The meaningful words are "excellent," "boring," "recommended."
Text analysis is like being a librarian organizing documents: break into words (tokenization), ignore filler words (stop word removal), count important words (frequency analysis), and use patterns to find specific info like phone numbers (regex).
import re

# ============ STRING METHODS ============
text = "  Hello, World! Welcome to Python.  "

print(text.strip())       # Remove leading/trailing whitespace
print(text.lower())       # "  hello, world! welcome to python.  "
print(text.upper())       # "  HELLO, WORLD! WELCOME TO PYTHON.  "
print(text.replace("World", "Data Science"))
print(text.split())        # Split by whitespace → list of words
print(", ".join(["a", "b", "c"]))  # "a, b, c"

# ============ REGULAR EXPRESSIONS ============
text = "Contact us at info@company.com or call 555-123-4567"

# Find all email addresses
emails = re.findall(r'[\w.]+@[\w.]+', text)
print(emails)  # ['info@company.com']

# Find all phone numbers
phones = re.findall(r'\d{3}-\d{3}-\d{4}', text)
print(phones)  # ['555-123-4567']

# Search for a pattern
match = re.search(r'\d+', "Order #12345")
print(match.group())  # "12345"

# ============ WORD FREQUENCY ============
from collections import Counter

text = "the cat sat on the mat the cat likes the mat"
words = text.lower().split()
freq = Counter(words)
print(freq.most_common(3))
# [('the', 4), ('cat', 2), ('mat', 2)]

# Remove stop words
stop_words = {"the", "on", "a", "an", "is", "in"}
filtered = [w for w in words if w not in stop_words]
print(Counter(filtered).most_common(3))
# [('cat', 2), ('mat', 2), ('sat', 1)]

⚠️ Common Mistakes & Misconceptions

Week 1: Python Basics

1. "Python is not case-sensitive"
Writing Print("hello") instead of print("hello"), or using True as true. Python IS case-sensitive — every character matters.
✓ Always use lowercase for built-in functions: print(), len(), type(). Boolean values are capitalized: True, False, None.
2. Concatenating strings and numbers directly
"I am " + 25 + " years old" → TypeError: can only concatenate str to str (FAQ #14)
✓ Convert with str(): "I am " + str(25) or use f-strings: f"I am {25} years old". F-strings handle conversion automatically.
3. Confusing = and ==
Using x = 5 when you mean to check equality x == 5. One assigns, the other compares.
✓ = is assignment (x = 5 stores 5 in x). == is comparison (x == 5 returns True/False). Use == in if statements.
4. Modifying a list while iterating over it
Removing items from a list inside a for loop causes skipped elements or IndexError.
✓ Create a new list with list comprehension: filtered = [x for x in items if x != "remove"]. Never modify a collection while looping through it.
5. Forgetting that range() excludes the stop value
range(1, 5) produces [1, 2, 3, 4], NOT [1, 2, 3, 4, 5]. Beginners expect 5 to be included.
✓ range(start, stop) goes from start to stop-1. To include 5: range(1, 6). Think of it as "up to but not including."

Week 2: Data Manipulation

6. Using df["col"] = df["col"].fillna(value) without inplace
Writing df["Age"].fillna(0) but forgetting to assign it back. fillna() returns a new Series by default.
✓ Either df["Age"] = df["Age"].fillna(0) or df["Age"].fillna(0, inplace=True). Always assign the result!
7. Using loc when you mean iloc (or vice versa)
df.loc[0:3] returns 4 rows (label-based, inclusive), but df.iloc[0:3] returns 3 rows (position-based, exclusive stop).
✓ loc = label-based (inclusive both ends). iloc = integer position-based (exclusive stop, like range()). When in doubt, use iloc for predictable slicing.

Week 3: EDA & Visualization

8. "Correlation means causation"
Seeing a high correlation between two variables and concluding one causes the other.
✓ Correlation only measures how variables move together. Always look for confounding variables. Ice cream sales and drownings correlate because both increase in summer — not because one causes the other.
9. Using mean for skewed data
Reporting mean salary as $80K when most employees earn $50K and a few executives earn $500K.
✓ For skewed distributions, use median (robust to outliers). Always visualize first — a histogram reveals whether mean or median is more representative.
10. Forgetting the raw string prefix in regex
re.findall("\bword\b", text) fails because \b is a backspace character in normal strings.
✓ Always use raw strings with regex: r"\bword\b". The r prefix tells Python to treat backslashes literally.
